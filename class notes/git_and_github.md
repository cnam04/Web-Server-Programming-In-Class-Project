# Git and Github

## Git
- Git is a protocol, there is no one program that *is* git
- Most programs use libraries from GitSCM and wrap them. 
## Source control
### What is source control?
- Think of it as the time dimension of files
- **Source control** allows you to track files over time, go back to previous version, look at changes
### Benefits of source control
- Allows multiple people to work collaboratively
- Serves as a tool to resolve conflicts between edits
    * Allows you to choose which edits to keep, or combine edits manually
- Allows you to create **feature branches**
    * preserve the working version of the application
    * you can work on the feature in that isolated environment
    * merge it back when ready
- **Commits:**
    * The way something is saved in git is called a commit
    * **Dont** wait to make a commit
    * The more commits, the better
- **Cloning:**
    * Allows you to get a copy of a repository
    * In each clone, you have multiple branches
    * Each clone is the same repository, but can possibly be a different version
- **Forking:**
    * Clones a repo to another repo on github
    * Allows you to work on a repo in a second repo
    * Especially useful when you don't have permissions to work in a repo
        * Then, you can make a pull request which notifies repo owners and allows you to send them your changes for approval
- **Issues:**
    * You can use this to keep track of things you want to accomplish
    * Other people can send issues
- **Readme:**
    * Readme is like the homepage for a folder in github
    * Provides a description of what the project, file, etc. does
